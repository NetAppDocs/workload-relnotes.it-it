== 03 maggio 2026



=== Amazon FSX per NetApp ONTAP



==== Cloni dei dispositivi a blocchi

Workload Factory offre cloni per i tuoi dispositivi a blocchi, così puoi fornire rapidamente copie scrivibili ed efficienti dei tuoi dati controllando capacità, accesso e ciclo di vita.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gestire storage a blocchi in NetApp Workload Factory"]



==== Tag EC2 sui dispositivi a blocchi

La tabella dei dispositivi a blocchi ora include una colonna che mostra i tag degli host EC2 che utilizzano ciascun dispositivo a blocchi. Questo consente di identificare quale host utilizza ciascun dispositivo a blocchi durante la risoluzione dei problemi o le verifiche.

Per creare tag su ciascun dispositivo a blocchi, distribuisci un'istanza EC2 utilizzando lo script utente che ti forniamo.

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["Script EC2 per l'assegnazione di tag ai dispositivi a blocchi"]



==== Andamento della capacità dei file system FSx for ONTAP

Scopri le tendenze della capacità SSD direttamente dalla panoramica dei tuoi file system FSx for ONTAP. La nuova scheda per la gestione della capacità offre una visione completa delle tendenze di utilizzo del mese precedente. Questa analisi include un confronto tra la capacità effettivamente utilizzata e la capacità allocata, consentendo una valutazione chiara dell'utilizzo delle risorse.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["Gestisci automaticamente la capacità in NetApp Workload Factory"]

=== Amazon FSX per NetApp ONTAP



==== Aggiornamenti di analisi ben progettati

Workload Factory analizza i file system FSx for ONTAP alla ricerca di *volumi NAS inattivi*. L'analisi della configurazione identifica i volumi NAS non utilizzati attivamente e suggerisce azioni per ottimizzare l'utilizzo dello storage e ridurre i costi.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Implementare FSx ben architettato per i file system ONTAP"]



==== Funzionalità della tabella di registro per i punti di accesso S3

La funzionalità Tabella di registro offre un modo per gestire FSx ONTAP S3 object storage su larga scala, consentendo di controllare facilmente gli eventi di accesso degli utenti e le operazioni sugli oggetti nei bucket. Con Workload Factory, è possibile configurare scansioni automatiche dei bucket e delle tabelle S3 per tenere traccia degli eventi, visualizzando tutti gli oggetti, i relativi dettagli e le azioni degli utenti.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["Configura l'infrastruttura della tabella di journaling per NetApp Workload Factory"]



==== Supporto per la gestione degli igroup esistenti

Workload Factory supporta la gestione di igroups esistenti. È possibile aggiungere o rimuovere dispositivi a blocchi e iniziatori host per consegnare il workload distribuito nei tempi previsti.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gestire storage a blocchi in NetApp Workload Factory"]



==== Supporto per la specifica dell'aggregato per i volumi FlexVol

Workload Factory supporta la creazione di volumi FlexVol su un aggregato selezionato. Ciò consente di suddividere il carico di capacità e prestazioni tra tutte le risorse del file system.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["Crea un FlexVol volume in NetApp Workload Factory"]



==== Ultimo orario di accesso per i dispositivi a blocchi

Workload Factory segnala l'ora dell'ultimo accesso per i dispositivi a blocchi. L'ora si basa sull'ultimo accesso effettuato all'igroup in Workload Factory. Conoscere l'ultima data di utilizzo di ciascun dispositivo a blocchi consente di ottimizzare le risorse.



=== Workload dei database



==== Analisi ben progettata per Oracle

L'analisi ben strutturata include le seguenti nuove configurazioni per Oracle:

* *Aggiornamenti critici delle patch Oracle*: verifica la presenza di patch di sicurezza mancanti per le installazioni di database Oracle autogestite supportate.
* *Replica tra regioni*: verifica se la replica tra regioni (CRR) è abilitata per i file system FSx for ONTAP che ospitano database Oracle.
* *Snapshot coerenti con l'applicazione*: verifica se il database Oracle è protetto da snapshot coerenti con l'applicazione con NetApp SnapCenter.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementare configurazioni di database ben progettate in Workload Factory"]



==== Valutazione una tantum per Oracle

Workload Factory offre ora una valutazione una tantum per analizzare le configurazioni delle tue implementazioni di database Oracle ogni volta che ne hai bisogno: nuove implementazioni, post-migrazione, risoluzione dei problemi e altro ancora. Riceverai consigli per aiutarti a ottimizzare le configurazioni delle tue implementazioni e a rispettare le best practice. Questa è la tua opportunità per utilizzare la funzionalità ben progettata senza dover completare l'intero processo di configurazione. Puoi valutare i tuoi database senza memorizzare le credenziali in Workload Factory, registrare risorse di database o configurare connessioni esterne.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Esegui una valutazione una tantum delle tue implementazioni di database Oracle"]



==== Risparmio sui costi migliorato per diversi database Oracle

Abbiamo migliorato l'analisi del risparmio sui costi per Oracle al fine di fornire una soluzione storage consolidata per diversi database Oracle on-premises su un unico file system FSx for ONTAP. Questo consolidamento ottimizza i costi di storage riducendo il numero di file system necessari per più database, portando a un maggiore risparmio sui costi.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Scopri le possibilità di risparmio per i database rilevati su Oracle on-premises"]

=== Carichi di lavoro EDA



==== Raccomandazioni sulla capacità EDA

Workload Factory ora analizza automaticamente la capacità SSD di ciascun cluster FSx for ONTAP presente nell'inventario EDA e genera raccomandazioni di capacità quando sono necessari aggiustamenti. L'algoritmo viene eseguito una volta ogni 24 ore per ciascun cluster.

Quando viene individuata una raccomandazione:

* Ricevi immediatamente un'email o una notifica WAD (non inclusa nel riepilogo settimanale).
* Nella tabella Cluster, sul cluster interessato compare un indicatore a forma di lampadina.
* Nella dashboard Panoramica è presente un riepilogo che mostra il numero totale di cluster con raccomandazioni attive.


Seleziona un cluster con una raccomandazione attiva per visualizzare la raccomandazione, che spiega la modifica suggerita e le relative motivazioni.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Scopri di più sulle raccomandazioni relative alla capacità"^].



==== Analisi automatizzata della latenza dell'AI

L'analisi della latenza ora include l'analisi degli agenti AI.

Per scenari complessi che coinvolgono dati o latenza del cluster, è possibile richiamare l'analisi dell'agente AI per ricevere:

* Identificazione dettagliata della causa principale
* Elenco dei client EC2 interessati
* Misure di risanamento specifiche


L'analisi tramite agente AI richiede la configurazione dell'ARN del modello Amazon Bedrock ed è disponibile su richiesta per gli scenari in cui l'analisi di base raccomanda un'indagine più approfondita. L'agente AI è in grado di identificare problemi quali volumi eccessivi, configurazioni non ottimali dei punti di montaggio, FlexGroup, necessità di ribilanciamento e requisiti di scale-out.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Scopri di più sull'analisi degli agenti AI"^].



== 09 marzo 2026



=== Workload VMware



==== Introduzione del supporto di migration advisor per Google Cloud VMware Engine

NetApp Workload Factory ora supporta Google Cloud VMware Engine. Puoi migrare i tuoi carichi di lavoro VMware on-premises su Google Cloud VMware Engine utilizzando Google Cloud NetApp Volumes come datastore NFS esterni, ottimizzando i costi e consentendo un maggiore controllo sul tuo ambiente VMware senza dover riorganizzare o riconfigurare le tue applicazioni.

Il consulente per la migrazione di Google Cloud VMware Engine fornisce:

* Dimensionamento e progettazione automatizzati dei cluster Google Cloud VMware Engine con supporto dei tipi di nodo VE1 e VE2
* Ottimizzazione del layout della VM tra i datastore interni vSAN e i datastore NFS esterni Google Cloud NetApp Volumes.
* Confronto dettagliato dei costi e analisi del TCO tra configurazioni solo vSAN e configurazioni potenziate con Google Cloud NetApp Volumes.
* Supporto per i livelli di servizio Standard, Premium ed Extreme di Google Cloud NetApp Volumes.
* Distribuzione Infrastructure-as-Code con Terraform per il provisioning automatico.

=== Amazon FSX per NetApp ONTAP



==== Aggiornamenti di analisi ben progettati

Workload Factory analizza i file system FSx for ONTAP per le seguenti configurazioni:

* Capacità SSD sottoutilizzata: verifica se il livello di capacità SSD è sottoutilizzato e consiglia di ridurre la capacità SSD per ottimizzare i costi.
* Pianifica backup dei volumi: NetApp Workload Factory migliora le raccomandazioni di configurazione e aggiunge opzioni per attivare il backup di FSx for ONTAP o AWS Backup. FSx for ONTAP backup applica una policy predefinita a tutti i volumi in un file system, creando backup giornalieri conservati fino a 90 giorni. AWS Backup consente di scegliere periodi di conservazione diversi per ciascun volume nel piano di backup.




==== Workload Factory supporta il caso d'uso AI & ML per la replica

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replicare i dati on-premises per casi d'uso di AI e ML"]

==== Aggiornamento della funzionalità di gestione automatica della capacità

Questo aggiornamento separa la funzionalità in due funzionalità distinte: gestione automatica della capacità e notifiche sulla capacità.

La gestione automatica della capacità aggiunge o rimuove automaticamente la capacità del file system per mantenere prestazioni ottimali. Le nuove opzioni includono modalità incrementale e adattiva. In modalità incrementale, il sistema aumenta la capacità di un importo fisso alle soglie; in modalità adattiva, utilizza dati storici per prevedere e regolare la capacità.

Le notifiche sulla capacità ti avvisano ogni volta che si verifica una modifica della capacità e quando i file system raggiungono i limiti impostati, così puoi gestire lo storage prima che si verifichino problemi.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["Scopri la gestione automatica della capacità e le notifiche di capacità in NetApp Workload Factory"]

=== Carichi di lavoro EDA



==== Dashboard panoramica per la gestione dello storage EDA

In qualità di amministratore IT che gestisce decine di file system FSx for ONTAP, puoi utilizzare la dashboard Panoramica per rispondere alle domande operative più comuni sullo stato del cluster, sull'utilizzo e sul posizionamento dei volumi.

La dashboard Panoramica include:

* *Stato di integrità del cluster*: informazioni riepilogate nella parte superiore della dashboard che evidenziano performance, capacità, disponibilità e protezione, ed eventi di sicurezza nei tuoi cluster.
* *Tabella dei cluster*: una tabella ricercabile, filtrabile e impaginata che mostra la capacità SSD, il pool di capacità e le metriche di throughput per cluster, con supporto per la personalizzazione delle colonne e l'esportazione CSV.


La precedente dashboard principale è ora accessibile nella scheda *Project*. Le schede EDA sono ora ordinate: Panoramica, Latenza, Project, CI/CD.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html["Scopri di più sulla dashboard Panoramica"^].



==== Analisi automatizzata della latenza

L'analisi della latenza ora include un'analisi di base automatizzata. Quando vengono rilevati eventi di latenza, il sistema analizza automaticamente le metriche del centro di ritardo QoS di ONTAP per identificare la fonte dei problemi di latenza.

L'analisi di base valuta le policy QoS, le prestazioni del disco, i problemi di rete e le configurazioni del cluster per fornire informazioni immediate nella colonna Dettagli della tabella degli eventi di latenza.

=== Amazon FSX per NetApp ONTAP



==== Supporto per la migrazione delle storage VM

NetApp Workload Factory ora supporta la migrazione per le storage VM. Questa funzionalità consente la migrazione dei dati e delle configurazioni del sistema storage ONTAP da sistemi ONTAP on-premises o da file system FSx for ONTAP di prima generazione a file system FSx for ONTAP di seconda generazione. È possibile replicare i dati e le impostazioni di configurazione delle storage VM per spostarsi nel nuovo file system con tempi di inattività e interruzioni minimi per utenti e applicazioni.

Per utilizzare questa funzionalità, link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["creare una relazione di replicazione"] e selezionare *Migrazione* come caso d'uso. Per completare il processo di migrazione, è necessario link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html#cut-over-replication-for-migration-use-cases["cutover della storage VM e dei suoi volumi replicati"] immediatamente per migrare in modo permanente i dati e le impostazioni di configurazione della storage VM nel file system FSx for ONTAP di destinazione.
