== 06 luglio 2026



=== Workload VMware



==== Supporto per l'istanza i7i.metal-24xl nel calcolatore TCO di EVS

Il calcolatore TCO di EVS ora supporta le istanze EC2 AWS i7i.metal-24xl come host del cluster. Rispetto alle configurazioni i4i.metal, queste istanze riducono il costo iniziale di EVS di circa 33% per il compute e 25% per lo storage vSAN.

Il calcolatore confronta automaticamente entrambi i tipi di istanza e seleziona l'opzione più conveniente. Una nuova sezione di confronto mostra le configurazioni degli host EVS affiancate, rendendo più facile trovare la configurazione migliore per le tue esigenze.

link:calculate-evs-savings.html["Calcola il risparmio stimato per EVS"]

=== Workload dei database



==== Risparmio personalizzato per gli host Oracle su Amazon Elastic Block Store

NetApp Workload Factory for Databases consente di personalizzare il risparmio per gli host Oracle su Amazon Elastic Block Store (EBS). È possibile regolare diversi campi, come la frequenza degli snapshot, il numero di copie clonate e i dettagli del volume, per verificarne l'impatto sul risparmio.

==== Selezione di più gruppi di sicurezza per la distribuzione su host Microsoft SQL Server e PostgreSQL

NetApp Workload Factory for Databases ora consente di selezionare più gruppi di sicurezza durante la distribuzione di host Microsoft SQL Server e PostgreSQL. Ciò offre maggiore flessibilità nella configurazione della sicurezza di rete per i carichi di lavoro del database.

* link:https://docs.netapp.com/us-en/workload-databases/create-database-server.html["Distribuisci host Microsoft SQL Server"]
* link:https://docs.netapp.com/us-en/workload-databases/create-postgresql-server.html["Distribuisci host PostgreSQL"]

=== Amazon FSX per NetApp ONTAP



==== Supporto per AWS GovCloud (US) in NetApp Workload Factory

NetApp Workload Factory supporta gli account AWS GovCloud (US). Utilizzando un account GovCloud, è possibile distribuire e gestire carichi di lavoro di archiviazione nelle regioni AWS GovCloud (US).



==== Imposta le soglie di latenza del volume in NetApp Workload Factory

È possibile configurare soglie di latenza per singolo volume al fine di monitorare le prestazioni e ricevere avvisi quando i valori di latenza superano i limiti definiti. Workload Factory monitora sia la latenza in lettura che quella in scrittura e genera avvisi quando una delle due soglie viene superata.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["Imposta le soglie di latenza del volume in NetApp Workload Factory"]



==== Monitora i ritardi di replica in NetApp Workload Factory

Con NetApp Workload Factory, puoi monitorare i ritardi di replica per i file system FSx for ONTAP configurati con replica remota. I ritardi di replica indicano il ritardo temporale tra quando i dati vengono scritti nel file system di origine e quando vengono replicati nel file system di destinazione. Il monitoraggio dei ritardi di replica ti aiuta a controllare lo stato della configurazione di replica e a identificare potenziali problemi che potrebbero influire sulla coerenza dei dati e sugli obiettivi del punto di ripristino (RPO).

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-replication-delays.html["Monitora i ritardi di replica in NetApp Workload Factory"]



==== Identificazione dell'host e definizione del carico di lavoro in NetApp Workload Factory

Workload Factory identifica gli host che utilizzano lo storage e i tipi di carichi di lavoro associati all'attività degli host. Le definizioni dei carichi di lavoro vengono generate dall'intelligenza artificiale in base ai tag AWS e ai metadati. È possibile visualizzare i dettagli degli host e le definizioni dei carichi di lavoro dalle schermate delle risorse relative a file system, volumi e dispositivi a blocchi in Workload Factory.



==== Dettagli LIF disponibili in NetApp Workload Factory

Visualizziamo tutti gli endpoint del file system e delle VM di storage direttamente nella scheda Panoramica per un file system. La visibilità diretta degli endpoint all'interno della console di Workload Factory semplifica il monitoraggio dell'accesso alla rete, degli endpoint e dello stato del failover senza utilizzare la CLI o CloudShell. Per visualizzare i dettagli di LIF, inclusi ruoli, endpoint, nodi, indirizzi IP e stato, è necessario associare un collegamento al file system.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["Scopri come associare un collegamento al tuo file system in NetApp Workload Factory"].



==== Miglioramenti dei costi

Sono disponibili nuovi filtri opzionali a livello di pagina per migliorare i risultati dell'analisi dei costi delle risorse in tutti gli account AWS. È possibile selezionare più regioni e utilizzare la chiave e i valori chiave del tag di allocazione del file system per filtrare i risultati dell'analisi dei costi e ottenere dettagli sui costi specifici delle risorse per i file system e i volumi FSx for ONTAP.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-costs.html["Utilizzare i filtri per l'analisi dei costi in NetApp Workload Factory"]



==== Pianificazione dell'inizializzazione della replica disponibile in NetApp Workload Factory

Durante la configurazione della replica dei dati, è ora possibile pianificare l'inizializzazione della replica per i file system FSx for ONTAP. Le opzioni sono di inizializzare la relazione di replica immediatamente o in un momento specificato. L'inizializzazione della replica è il processo di creazione di una baseline per la replica tramite la copia dei dati dal file system di origine al file system di destinazione. La pianificazione dell'inizializzazione della replica consente di automatizzare questo processo e garantire che la configurazione della replica sia aggiornata senza intervento manuale.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replicare i dati in NetApp Workload Factory"]



==== Report Excel sullo stato di corretta architettura dei file system FSx for ONTAP

Dopo l'analisi, è possibile scaricare un report Excel sullo stato di corretta architettura dei file system FSx for ONTAP per i propri archivi o da condividere con i team.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Visualizza lo stato di corretta architettura dei tuoi file system FSx for ONTAP in NetApp Workload Factory"]



==== Visualizza e analizza gli eventi di latenza

Workload Factory ora supporta la visualizzazione e l'analisi degli eventi di latenza per i volumi FSx for ONTAP. Puoi visualizzare gli eventi di latenza senza una connessione link, rendendo più semplice il monitoraggio dei problemi di prestazioni. Per analizzare gli eventi di latenza e ricevere approfondimenti e raccomandazioni basati sull'intelligenza artificiale, è necessario connettersi al file system tramite un link. L'analisi fornisce i valori di latenza attuali, le soglie, l'analisi della risposta QoS e le indicazioni per la risoluzione.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/analyze-events.html["Analizza gli eventi FSx for ONTAP in NetApp Workload Factory"]



==== Imposta le soglie di latenza del volume

Ora è possibile configurare le soglie di latenza per singolo volume, al fine di monitorare le prestazioni e ricevere avvisi quando la latenza di lettura o scrittura supera i limiti definiti. Il monitoraggio della latenza consente di identificare in modo proattivo i problemi di prestazioni prima che abbiano un impatto sui carichi di lavoro.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["Imposta le soglie di latenza del volume in NetApp Workload Factory"]

=== Carichi di lavoro EDA



==== Modalità di gestione dell'aumento della capacità SSD e automazione

Workload Factory ora offre modalità di gestione per l'aumento della capacità SSD in base alle tue preferenze operative:

* *Automatizzare*
* *Consiglia*
* *Nessuno*


La modalità corrente e i relativi parametri sono visualizzati nella parte superiore del grafico di utilizzo dell'SSD nella pagina dei dettagli del file system. È possibile cambiare modalità in qualsiasi momento in base alle proprie esigenze operative.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#change-ssd-management-mode["Scopri le modalità di gestione degli SSD"^].



==== Modifica in blocco dei parametri di gestione per l'aumento della capacità SSD

Ora è possibile regolare i parametri di gestione dell'aumento della capacità SSD su più file system dalla dashboard Panoramica. Seleziona uno o più cluster nella tabella Cluster e modifica in blocco i valori di Threshold (10-90%), Lookback (1-200 ore) e Ahead (1-200 ore).

La finestra di dialogo di modifica si popola automaticamente in base alla selezione effettuata: se tutti i file system selezionati condividono gli stessi valori dei parametri, questi vengono visualizzati; in caso contrario, vengono utilizzati i valori predefiniti. Ciò consente di applicare impostazioni di capacità coerenti in tutto l'ambiente.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#bulk-edit-ssd-management-parameters["Scopri come modificare i parametri in blocco"^].



==== Visualizzazione dei punti decisionali di capacità

Il grafico di utilizzo degli SSD ora mostra i punti decisionali che indicano quando sono state generate raccomandazioni sulla capacità o sono state intraprese azioni automatiche.

* *Punti decisionali consigliati*: Passa il mouse sopra per visualizzare data e ora, dimensione di aumento consigliata e capacità attuale.
* *Punti decisionali dell'automazione*: Passa il mouse sopra per visualizzare data e ora, stato di successo o fallimento, aumentare la dimensione e la nuova dimensione dell'aggregato. I tentativi falliti includono la motivazione del fallimento.


I punti decisionali possono verificarsi ogni 30 minuti e vengono visualizzati singolarmente o in forma aggregata a seconda dell'intervallo di tempo selezionato.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#analyze-capacity-trends-and-decision-points["Scopri i punti di decisione della capacità"^].



==== Stato di configurazione del collegamento nella tabella Cluster

La tabella Cluster nella dashboard Panoramica ora include una colonna Collegamento associato che indica se un collegamento FSx for ONTAP è configurato per ciascun file system. Lo stato della connessione viene convalidato quotidianamente, consentendo di identificare i file system che necessitano di configurazione del collegamento o che potrebbero presentare problemi di connettività.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html["Visualizza la dashboard Panoramica"^].

=== Workload dei database



==== Analisi ben progettata per Oracle

L'analisi ben strutturata include le seguenti nuove configurazioni per Oracle:

* *Configurazione del backup*: verifica se i database Oracle sono protetti da AWS Backup o da FSx for ONTAP backup.
* *Pulizia dei cloni*: si consiglia di rimuovere i cloni obsoleti o inutilizzati per ridurre i costi di storage.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementare configurazioni di database ben progettate in Workload Factory"]



==== Calcolo automatico dei risparmi per i database Oracle su Amazon Elastic Block Store (EBS)

Workload Factory analizza automaticamente il costo dello storage EBS utilizzato per i database Oracle e quindi suggerisce la configurazione FSx for ONTAP migliore e più conveniente. È possibile selezionare più host EBS da analizzare e quindi confrontare il costo di esecuzione di tali host su EBS rispetto a FSx for ONTAP.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Esplora i risparmi per gli host EBS rilevati"]



==== Verifica dei prerequisiti di configurazione CRR

Quando si abilita la replica tra regioni (CRR) per il database estate, il carico di lavoro Database verifica se il file system FSx for ONTAP dispone di un collegamento associato e di una configurazione di replica. Se necessario, il carico di lavoro Database reindirizza l'utente alla procedura guidata di replica del carico di lavoro Storage di FSx for ONTAP per gestire i prerequisiti del collegamento nello stesso modo in cui li gestisce Storage.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementare configurazioni di database ben progettate in Workload Factory"]



==== LUN associati per i database di Microsoft SQL Server

Ora è possibile visualizzare le LUN mappate ai database di Microsoft SQL Server in Workload Factory dalla schermata Inventario. Ciò consente di identificare facilmente quali LUN sono associate a quali database, semplificando la gestione e l'ottimizzazione delle risorse di storage.



=== Installazione e amministrazione



==== Notifiche e-mail per Workload Factory

Ora puoi ricevere le notifiche di Workload Factory via email senza dover configurare un argomento Amazon SNS. Abilita le notifiche email per inviarle a tutti gli indirizzi email associati all'account.

È possibile configurare le notifiche e-mail sia dalla console di Workload Factory che dalla NetApp Console. La NetApp Console offre anche la possibilità di configurare le notifiche della NetApp Console che vengono visualizzate sotto l'icona a forma di campana.

Se si configurano le notifiche nella NetApp Console, NetApp Console assume il controllo della distribuzione delle notifiche. Le impostazioni di notifica nella console di Workload Factory diventano di sola lettura per evitare notifiche duplicate.

=== Carichi di lavoro EDA



==== Notifiche e-mail e SNS per eventi di latenza

Ora puoi ricevere notifiche automatiche quando vengono rilevati eventi di latenza sui tuoi volumi FSx for ONTAP. Configura le notifiche via email o Amazon SNS nelle impostazioni di Workload Factory per ricevere avvisi in real-time quando i volumi superano le soglie di latenza configurate.

Le notifiche vengono inviate a livello di file system e includono dettagli su tutti i volumi interessati, i livelli di gravità, i confronti con le soglie e un collegamento diretto alla pagina Latenza per l'indagine.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["Scopri come configurare le notifiche di latenza"^].



==== Visualizzazione dell'andamento della latenza con grafici interattivi

Il pannello di analisi della latenza ora include un'interfaccia a schede per un'analisi completa della latenza:

* *Panoramica*: Visualizza i risultati dell'analisi di base automatizzata che mostrano quale componente sta causando latenza
* *Nel tempo*: Visualizza l'andamento della latenza del volume con un grafico interattivo su più intervalli di tempo (1 ora, 3 ore, 12 ore, 24 ore o 72 ore)


La scheda *Nel tempo* visualizza i dati di latenza raw CloudWatch (lettura o scrittura, a seconda dell'allarme attivato) con linee di soglia di avviso e critiche, indicatori di violazione e metriche dettagliate per ogni violazione.

Questa visualizzazione completa ti aiuta a identificare schemi, a capire se i problemi di latenza sono ricorrenti o isolati e a prendere decisioni più informate sulle azioni correttive.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["Scopri come analizzare le tendenze della latenza"^].
